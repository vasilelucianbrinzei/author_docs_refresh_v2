# Set Up GitHub and Install Tools

## Introduction

This lab prepares your workstation for LiveLabs authoring. It covers the minimum setup most authors need before they fork repositories, edit Markdown, and preview workshops locally or through GitHub Pages.

Estimated Time: 20 minutes

### Objectives

* Set up a GitHub account for LiveLabs work
* Install GitHub Desktop
* Install Visual Studio Code and a local preview extension
* Add a small set of useful editor extensions for workshop authoring

### What Do You Need?

* An Oracle email address if you are contributing as an Oracle employee

## Task 1: Create And Prepare Your GitHub Account

1. Create a GitHub account if you do not already have one.
2. Use your Oracle email address for the account if you are an Oracle employee. Do not create a second account just for LiveLabs work.
3. Update your GitHub profile so your name and picture are recognizable to reviewers and repository owners.
4. Enable two-factor authentication on the account before you start contributing.
5. If you are an external contributor, confirm the correct repository path and contributor requirements before you begin authoring.

## Task 2: Install GitHub Desktop

1. Download and install **GitHub Desktop**.
2. Sign in with the GitHub account you will use for LiveLabs work.
3. Confirm GitHub Desktop opens successfully and can see your repositories.

## Task 3: Install Visual Studio Code

1. Download and install **Visual Studio Code** for your operating system.
2. Launch VS Code and confirm it opens local folders correctly.
3. Use VS Code as the recommended editor for LiveLabs Markdown, manifest, and image-path work.

## Task 4: Install A Local Preview Extension

1. Open the **Extensions** panel in VS Code.
2. Install **Live Server** so you can preview workshop changes locally while you write.
3. Confirm the extension is available before you continue to the repository sync lab.

## Task 5: Add Optional But Useful Extensions

1. Configure Markdown indentation so nested content aligns correctly under numbered steps.
2. Install **markdownlint** to catch formatting issues while you write.
3. Install **Code Spell Checker** to reduce obvious spelling problems.
4. Install **Trailing Spaces** or an equivalent extension to remove whitespace issues that commonly break formatting.
5. Install **Path Intellisense** if you want help navigating relative file paths in `manifest.json` and Markdown links.
