# LiveLabs Start Guide

## Introduction

Use this page as the new entry point to the LiveLabs author guide. It now starts with an embedded interactive prototype built around two author modes:

* a guided path for new authors who need the shortest safe route from request to publish
* a reference toolkit for experienced authors who need to find a specific feature, tool, or QA pattern quickly

Use the interactive view first. If you need the classic documentation map afterward, use the section and background links lower on this page.

Estimated Time: 10 minutes

### Objectives

* Choose the right entry path for your author experience level
* Follow the shortest guided path from workshop request to publish
* Jump directly to the right section when you already know what you need
* Keep the full guide available as a fallback reference

## Task 2: Follow The Guided Path

1. Use the guided path when you want the minimum safe sequence:
    * **Step 1:** submit the workshop request in WMS
    * **Step 2:** create the workshop and add it to GitHub
    * **Step 3:** review, QA, and publish
2. After the request is approved, move into [Core Workflow](?lab=core-workflow) for the full execution details.
3. When the workshop is ready for formal validation and production handoff, use [Validation and Publish](?lab=validation-qa-publish).
