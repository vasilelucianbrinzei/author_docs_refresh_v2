# Need Help?

## Introduction

Use this page when you are blocked and need the fastest route to the right support channel. The goal is to solve authoring issues quickly without turning the main workflow into a support manual.

Estimated Time: 5 minutes

### Objectives

* Know which support channel fits which type of blocker
* Gather the right details before asking for help
* Avoid low-signal requests that slow down resolution

## Task 1: Confirm You Need Support

1. Check the section landing page for the topic you are working on so you do not ask a support question that the documentation already answers.
2. Review **LiveLabs FAQ** for common author and publishing questions.
3. Review **LiveLabs SLAs** if your question is about expected response times or handoff timing.
4. If the issue is tied to a non-standard environment or specialized flow, check the **Specialized Workflows and Reference** section before escalating.

## Task 2: Use The Right Support Channel

1. Use **WMS Message the Team** for workshop-specific questions about approval, stakeholder review, status changes, or publishing context tied to one workshop record.
2. Email `livelabs-help-db_us@oracle.com` for shared documentation, platform, tooling, or publishing support issues that are not owned by one workshop team.
3. Use the Slack channel `#workshop-authors-help` for quick unblockers, shared visibility, and questions that may help other authors too.
4. Use LiveLabs office hours when they are announced and the issue is easier to solve live than through async back-and-forth.

## Task 3: Ask For Help Efficiently

1. Include the WMS ID, repository name, branch or pull request, and preview URL when they are relevant.
2. State the exact blocker, what you expected to happen, and what you already tried.
3. Attach screenshots or error text when the issue is visual or tool-specific.
4. Do not send vague requests such as "it doesn't work" without context if you already have the error, failing step, or affected URL.
5. Return to the relevant workflow lab after the blocker is resolved so your workshop stays aligned with the documented process.
