# Get started with the LiveLabs AI Developer Hub

## Introduction

Use this page when you want AI-assisted help for drafting, restructuring, or accelerating a LiveLabs authoring task without losing alignment with the canonical workshop guide.

The AI Developer Hub is most useful when you already have source material and want help turning it into cleaner steps, tighter prose, or a first-pass structure faster. It is not a replacement for the canonical guide, workshop preview, or validator.

Estimated Time: 10 minutes

### Objectives

* Know where to open the AI Developer Hub guide, repo, and skill bundle
* Use AI assistance for narrow, real authoring tasks
* Keep AI-generated output grounded in the canonical guide, screenshots, preview, and validator checks

## Task 1: Start With The Published Hub Entry Points

1. Open the [LiveLabs AI Developer Hub guide](https://lfoinding.github.io/livelabs-ai-playground/skills/how-to/workshops/sandbox/index.html) and review the intended workflow before you start prompting.
2. Open the [LiveLabs AI Developer Hub repository](https://github.com/lfoinding/livelabs-ai-playground) if you want the materials locally.
3. Review the [LiveLabs AI Developer skill bundle](https://github.com/lfoinding/livelabs-ai-playground/tree/main/LiveLabs-AI-Developer) so you know which reusable authoring helpers already exist.

## Task 2: Use AI For Real Authoring Work, Not Vague Summaries

1. Give AI one focused task at a time, such as:

    * Drafting step-by-step task instructions from source content
    * Tightening prose in an existing lab
    * Extracting prerequisites, commands, or checkpoints
    * Restructuring a section so the flow is easier to follow

2. Start from the canonical guide or workshop content you already trust instead of asking AI to invent the workflow from scratch.
3. Keep asking for exact steps, commands, field examples, and screenshots to reference when the context supports them.

## Task 3: Validate Everything Before You Keep It

1. Compare AI-assisted output against the canonical guide and keep only the parts that are accurate.
2. Reopen the workshop preview and make sure the updated steps still match the UI, paths, and screenshots the learner will actually see.
3. Run the validator before you treat the AI-assisted draft as complete.
4. If the output stays vague or drifts away from the source guide, tighten the prompt or return to the manual workflow.

## Task 4: Clone The Repo If You Want The Materials Locally

```bash
git clone https://github.com/lfoinding/livelabs-ai-playground.git
```
