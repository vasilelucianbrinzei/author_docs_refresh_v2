# Tools and Productivity

## Introduction

Use this section for the practical tools that improve screenshot quality, reduce asset size, and speed up cleanup. These labs support the main author workflow instead of replacing it, and they work best when you use them in the same order that polish problems appear.

Estimated Time: 5 minutes

## Objectives

* Know which tool to use for each common authoring problem
* Use screenshot and cleanup tools at the right point in the workflow
* Keep the tool choices connected to review, QA, and publish needs

## Task 1: Use The Tools In A Clean Order

1. Capture clean screenshots first.
2. Optimize image size second.
3. Use automation-based cleanup only after the content is already structurally stable.

## Task 3: Keep Tools Connected To The Workflow

1. Use these labs while you are developing or polishing content, not as a separate disconnected track.
2. Return to **Core Workflow -> Lab 6** after tool-driven changes if you need to update manifests, Markdown, or images.
3. Move into **Validation and Publish** once the screenshots and cleanup work are done.

## Task 4: Before You Leave This Section

Use this checkpoint before you move on:

* Screenshots are cropped to the action, readable, and safely redacted.
* Large images are reduced before the PR checks complain about them.
* Cleanup automation results were reviewed instead of accepted blindly.
* The workshop still previews correctly after the image or markdown cleanup pass.

## Task 5: Common Tooling Mistakes

* Trying to fix a badly captured screenshot only through compression or resize.
* Running optimization or cleanup against the wrong folder.
* Treating image cleanup as unrelated to the actual author workflow.
