# Reuse and Enhancements

## Introduction

Use this section for optional additions that extend a workshop after the core author path is already in place. These labs are not required for every workshop, and they should be added only when they improve the learner flow clearly enough to justify the extra complexity.

Estimated Time: 5 minutes

## Objectives

* Identify optional enhancements that fit your workshop
* Know when a reusable component is worth the added complexity
* Keep optional components separate from the required author workflow

## Task 2: Keep Enhancements In Proportion

1. Finish the core author path before you add optional components.
2. Add only the components that improve comprehension or task completion.
3. After integrating a component, return to **Validation and Publish** so the workshop is reviewed as a complete experience.

## Task 3: Before You Keep An Enhancement

Use this checkpoint before you keep an optional component in the workshop:

* The component supports the task right where the learner needs it.
* The surrounding instructions still make sense without forcing the learner to guess what to do next.
* The component is previewed and rendered correctly in the workshop flow.
* The enhancement is helping comprehension, not acting like disconnected novelty.

## Task 4: Common Misuse Patterns

* Adding FreeSQL when a normal copy-ready code block would be clearer.
* Adding quizzes far away from the step they are supposed to reinforce.
* Introducing optional features before the core markdown, media, and review flow are already stable.
