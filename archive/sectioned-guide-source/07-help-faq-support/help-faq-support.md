# Help and FAQ

## Introduction

Use this section when you need answers, escalation paths, or quick support while developing, reviewing, or maintaining a workshop. Use it to recover momentum without replacing the main workflow.

Estimated Time: 5 minutes

## Objectives

* Know where to go for practical support
* Know which page answers which type of question
* Know when to use FAQ versus direct support
* Know when AI Developer Hub can accelerate authoring without replacing the canonical workflow

## Task 1: Use Support In The Right Order

1. Check the FAQ first for repeat questions.
2. Use direct support only when the issue is specific, blocked, or ownership-related.
3. Use AI assistance for drafting or restructuring only after you know which canonical workflow it must stay aligned to.

## Task 3: Use Support Without Losing Momentum

1. Check the **LiveLabs FAQ** first for common questions.
2. Use **Need Help?** when you are blocked, need the right owner, or need to escalate.
3. Use **LiveLabs AI Developer Hub** when the goal is drafting, restructuring, or automating part of the author workflow faster, but verify the output against the canonical guide, preview, and validator.
4. If your question is about review timing or publishing handoffs, go back to **Validation and Publish** and use **Lab 8: Review, QA, and publishing timelines** there.

## Task 4: Bring This Context When You Ask For Help

Use this checklist so support can route or solve the issue faster:

* WMS ID
* Preview URL
* Repo, branch, or pull request link
* Current workshop status
* Exact blocker
* What you already tried

## Task 5: Common Support Misuse Patterns

* Asking for help before checking the FAQ or the relevant workflow lab.
* Sending vague support requests without links, IDs, or blocker details.
* Treating AI output as final without checking it against the canonical guide and preview.
